SAP ABAP Agent
cho Claude Code

Plugin Claude Code + MCP Server tự động kết nối SAP BTP / S/4HANA Cloud để thao tác ABAP — đọc, tìm, syntax-check, activate. Hỗ trợ multi-profile cho nhiều project SAP cùng lúc.

⚡ Bắt đầu cài đặt ➕ Thêm project SAP 📂 GitHub

🏠 Giới thiệu

SAP ABAP Agent là plugin dành cho Claude Code (Anthropic) kết hợp với MCP server, cho phép bạn tương tác trực tiếp với SAP BTP hoặc S/4HANA Cloud ngay trong terminal.

Bạn có thể đọc source code ABAP, tìm kiếm object, syntax-check, activate object — tất cả đều thông qua hội thoại với Claude mà không cần rời khỏi editor.

Điểm mạnh nhất là hỗ trợ multi-profile: mỗi project SAP (URL, tenant, OAuth2 riêng) là một profile độc lập, được lưu an toàn trong máy bạn với secret được mã hóa.

📦 Cài đặt

Yêu cầu

  • Node.js >= 18
  • Claude Code (Anthropic CLI)
  • Tài khoản SAP BTP / S/4HANA Cloud với OAuth2 client

Các bước cài đặt

  1. Clone repository và cài dependencies:
# Clone & cài MCP Server
cd sap-abap-agent
cd mcp-server
npm install
npm link

# Kiểm tra
sap-btp-agent --help
  1. Đăng ký MCP server với Claude Code:
// ~/.claude/mcp_servers.json (hoặc %USERPROFILE%\.claude\mcp_servers.json trên Windows)
{
  "mcpServers": {
    "sap-btp": {
      "command": "sap-btp-agent",
      "args": []
    }
  }
}

Sau bước này, các MCP tool như sap_ping, sap_search, sap_list_packages... sẽ xuất hiện trong Claude Code.

➕ Thêm project SAP mới

Cách nhanh nhất — truyền URL trực tiếp:

sap-btp-agent setup https://project1.s4hana.cloud.sap

Wizard sẽ tự sinh profile ID từ hostname (vd: project1.s4hana.cloud.sap) và hỏi:

  • OAuth2 client_id + client_secret (hoặc username/password, hoặc bearer token)
  • Region, service type

Thông tin được lưu riêng trong thư mục profile:

📂 %USERPROFILE%\.sap-btp-agent\profiles\project1.s4hana.cloud.sap\
├──
config.json ← URL, tenant, client_id, region, service (không nhạy cảm)
└──
secrets.json ← client_secret / token (ĐÃ MÃ HÓA)

Thêm project thứ 2, thứ 3... cũng dễ dàng:

sap-btp-agent setup https://project2.s4hana.cloud.sap
sap-btp-agent setup https://project3.s4hana.cloud.sap

Mỗi project có profile riêng, secret riêng (mã hóa độc lập).

👥 Quản lý nhiều profile

sap-btp-agent profiles list
Liệt kê tất cả profile (* = active)

Chọn profile active
sap-btp-agent profiles show
Xem chi tiết profile đang dùng

🔐 OAuth2 — Kết nối SAP BTP

SAP ABAP Agent hỗ trợ 3 chế độ xác thực OAuth2 để kết nối SAP BTP / S/4HANA Cloud. Tùy vào cấu hình tenant, bạn chọn chế độ phù hợp.

🔑

Client Credentials

Dùng client_id + client_secret. Phù hợp cho M2M automation, không cần user tương tác.

👤

Password Grant

Dùng username + password. Phù hợp khi bạn có tài khoản SAP BTP cá nhân.

🎫

Bearer Token

Dùng token có sẵn (nhập tay). Phù hợp cho testing hoặc token ngắn hạn.

Luồng hoạt động

Khi bạn gọi một MCP tool (vd sap_search), hệ thống tự động:

1. Kiểm tra token trong bộ nhớ (memory cache) 2. Nếu hết hạn → kiểm tra secret đã mã hóa trong secrets.json 3. Nếu còn hạn → dùng lại (tránh gọi OAuth2 liên tục) 4. Nếu hết hạn → gọi OAuth2 endpoint để lấy token mới 5. Tự động lưu refresh_token + expiresAt vào secrets.json 6. Token được refresh 30 giây trước khi hết hạn (SAFETY_MARGIN_S)

Cấu hình OAuth2 Client trên SAP BTP

Trước khi setup, bạn cần tạo OAuth2 client trên SAP BTP cockpit:

  1. Đăng nhập SAP BTP Cockpit → vào Subaccount của bạn
  2. Vào menu Security > OAuth2 Client (hoặc API Clients trong các phiên bản mới)
  3. Tạo client mới với Grant Type = Client Credentials
  4. Ghi lại Client ID và Client Secret
  5. Nếu dùng IAS (Identity Authentication), lưu ý token URL là /oauth2/token thay vì /oauth/token

Wizard Setup chi tiết

Khi chạy sap-btp-agent setup, wizard sẽ hỏi từng thông tin:

# Bước 1: Nhập URL SAP BTP

  3. Bearer token (nhập tay)

? Client Secret: ****************
? Scope (để trống nếu không cần):

# Bước 3b: Nếu chọn Password grant

# Bước 3c: Nếu chọn Bearer token

Token URL & Xử lý các hệ thống khác nhau

Mỗi hệ thống SAP có endpoint OAuth2 khác nhau. Agent tự động xác định dựa trên URL gốc, nhưng bạn có thể override nếu cần:

Hệ thống Token URL mặc định Ghi chú
XS UAA (cf) {btpUrl}/oauth/token Mặc định cho hầu hết S/4HANA Cloud
IAS {btpUrl}/oauth2/token Identity Authentication Service
Custom Override trong secrets.json Sửa tokenUrl theo ý muốn

Override Token URL thủ công

Nếu gặp lỗi 404 /oauth/token, bạn có thể sửa trực tiếp file secrets.json:

# Mở file secrets.json (đã mã hóa, cần giải mã qua CLI)
# HOẶC chạy lại setup với đúng token URL

# Cách nhanh: chạy lại setup với flag --overwrite
sap-btp-agent setup https://project1.s4hana.cloud.sap

# Sau đó sửa tokenUrl trong profile
# File nằm ở:
%USERPROFILE%\.sap-btp-agent\profiles\project1.s4hana.cloud.sap\secrets.json

Token Lifecycle & Auto-Refresh

Agent quản lý vòng đời token hoàn toàn tự động:

🔄 Tự động refresh

Token được cache trong bộ nhớ. Khi còn 30 giây trước khi hết hạn, agent tự động gọi OAuth2 endpoint để lấy token mới — bạn không cần làm gì cả.

💾 Lưu token an toàn

Token mới + refresh_token + expiresAt được lưu vào secrets.json đã mã hóa. Khi restart agent, token được phục hồi từ file mà không cần xác thực lại.

🔐 Mã hóa đa nền tảng

Windows: DPAPI (gắn với tài khoản Windows). macOS/Linux: AES-256-GCM, key derive từ hostname + username qua scrypt. File mode 0o600.

🍪 Cookie Auth & Web Popup Re-Auth

Bên cạnh OAuth2, SAP ABAP Agent hỗ trợ cookie-based authentication sử dụng SAP session cookies (MYSAPSSO2, SAP_SESSIONID, sap-usercontext). Phương thức này đặc biệt hữu dụng khi bạn đã có sẵn session từ SAP GUI, Fiori Launchpad, hoặc muốn tận dụng cơ chế Single Sign-On (SSO).

⚡ Tính năng chính

🍪

Cookie-Based Auth

Dùng session cookies SAP thay vì OAuth2 tokens. Hỗ trợ MYSAPSSO2, SAP_SESSIONID, v.v.

🖥️

Web Popup Re-Auth

Khi session hết hạn (401), tự động mở browser cho user đăng nhập lại — lấy cookies mới, tiếp tục công việc.

🛡️

Stampede Protection

Nhiều request cùng bị 401 chỉ kích hoạt 1 lần re-auth (mutex + cooldown 5s) — giống cơ chế reauthMu của vibing-steampunk.

🔒

Lưu cookies an toàn

Cookies được mã hóa và lưu trong secrets.json (DPAPI / AES-256-GCM), phục hồi khi restart agent.

🔧 CLI Wizard — Cài đặt Cookie Auth

Khi chạy sap-btp-agent setup, wizard sẽ hỏi bạn chọn phương thức xác thực. Chọn option 4 (Cookie-based):

# Chạy setup wizard
$ sap-btp-agent setup https://project1.s4hana.cloud.sap

# Wizard hỏi auth mode — chọn 4
? Chọn auth mode:
  1. OAuth2 (client_credentials) ← Mặc định
  2. Password (username/password)
  3. Bearer token (nhập tay)
  4. Cookie-based ← MỚI!

# Nhập cookies từ file hoặc paste tay
? Lay cookies tu: (1) File Netscape format  (2) Nhap tay
  → Chọn 2 và paste cookie string từ browser

? Cookie string (name=value; name2=value2):
  MYSAPSSO2=abc123...; SAP_SESSIONID=xyz456...

# Chọn chế độ re-auth
? Che do re-auth khi session het han?
  (1) Manual paste  ← Mở browser, user login, paste cookie
  (2) Auto (Playwright) ← Tự động mở + trích xuất cookies

# Hoàn tất
✅ Da tao profile 'project1.s4hana.cloud.sap' thanh cong!
  i Auth mode: cookie
  i Re-auth mode: manual
  i Nhan dien 3 cookies (MYSAPSSO2, SAP_SESSIONID, sap-usercontext)

🌐 Web Popup Re-Auth — Khi Session Hết Hạn

Khi đang code mà session SAP hết hạn (HTTP 401), agent tự động:

1. Phát hiện 401 Unauthorized từ SAP 2. Kích hoạt stampede guard (chỉ 1 lần re-auth dù nhiều request cùng fail) 3. Gọi ReauthHandler — mở trình duyệt web 4. Hiển thị thông báo "🔐 SESSION SAP DA HET HAN!" + URL đăng nhập 5. User đăng nhập vào SAP qua browser 6. Manual mode: User copy cookies từ DevTools (F12) → paste vào terminal 7. Auto mode: Playwright tự động trích xuất cookies sau login 8. Cookies mới được lưu vào secrets.json (mã hóa) 9. Request được retry tự động với cookies mới → thành công!

🔐 Hai Chế Độ Re-Auth

👆

Manual Paste (Mặc định)

Dùng webbrowser.open() mở SAP URL. User login thủ công, sau đó mở DevTools (F12) → Application → Cookies → copy cookie string → paste vào terminal.

✅ Không cần cài đặt thêm. Hoạt động trên mọi nền tảng.

🤖

Auto (Playwright)

Dùng playwright mở Chromium, cho user login, tự động phát hiện khi đăng nhập xong và trích xuất cookies.

⚠️ Yêu cầu pip install playwright. Tự động fallback về manual nếu chưa cài.

📦 Cài đặt Playwright (cho Auto mode)

# Cài playwright + Chromium
pip install playwright
playwright install chromium

# Khi setup, chọn re-auth mode = Auto
sap-btp-agent setup https://project1.s4hana.cloud.sap
# → Chọn option 4 (Cookie), re-auth: 2 (Auto)

🧠 Kiến trúc — Lấy cảm hứng từ vibing-steampunk

Cơ chế cookie re-auth được xây dựng dựa trên pattern từ vibing-steampunk (dự án ADT → MCP bridge bằng Go):

vibing-steampunk (Go) SAP ABAP Agent (Python)
Transport.ReauthFunc ReauthHandler (callback type)
reauthMu (mutex) ReauthStampedeGuard (asyncio.Lock)
reauthCooldown = 5s cooldown_s = 5.0
LoadCookiesFromFile() _load_cookies_from_file()
chromedp (CDP) playwright / webbrowser

⚠️ Lưu ý

🍪 Cookies cần có SAP session

Cần ít nhất một trong các cookies: MYSAPSSO2, SAP_SESSIONID, sap-usercontext, hoặc sap-contextid.

🔄 Round-trip an toàn

Khi re-auth, cookies cũ được giữ nguyên cho đến khi cookies mới được xác nhận. Stampede protection ngăn multiple re-auth đồng thời.

⚠️ Lỗi thường gặp

❌ 401 Unauthorized

Client_secret sai hoặc hết hạn. Chạy sap-btp-agent setup <profile-id> để cập nhật thông tin OAuth2.

⚠️ 404 /oauth/token

URL token sai. Vào profiles/<id>/secrets.json, sửa tokenUrl (vd: đổi /oauth/token thành /oauth2/token nếu dùng IAS).

🔒 Không giải mã được secret

Đổi máy / hostname. Chạy lại sap-btp-agent setup <profile-id> để tạo lại secret.

📭 Chưa có profile nào

Chạy sap-btp-agent setup <URL> để thêm project SAP đầu tiên.

🔌 MCP tool không xuất hiện

Kiểm tra mcp_servers.json đã đúng cấu trúc chưa. Đảm bảo sap-btp-agent có trong PATH.
