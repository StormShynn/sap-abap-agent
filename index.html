cho Claude Code

Plugin Claude Code + MCP Server tự động kết nối SAP BTP / S/4HANA Cloud để thao tác ABAP — đọc, tìm, syntax-check, activate. Hỗ trợ multi-profile cho nhiều project SAP cùng lúc.

🏠 Giới thiệu

SAP ABAP Agent là plugin dành cho Claude Code (Anthropic) kết hợp với MCP server, cho phép bạn tương tác trực tiếp với SAP BTP hoặc S/4HANA Cloud ngay trong terminal.

Bạn có thể đọc source code ABAP, tìm kiếm object, syntax-check, activate object — tất cả đều thông qua hội thoại với Claude mà không cần rời khỏi editor.

Điểm mạnh nhất là hỗ trợ multi-profile: mỗi project SAP (URL, tenant, OAuth2 riêng) là một profile độc lập, được lưu an toàn trong máy bạn với secret được mã hóa.

🔐 OAuth2 — Kết nối SAP BTP

SAP ABAP Agent hỗ trợ 3 chế độ xác thực OAuth2 để kết nối SAP BTP / S/4HANA Cloud. Tùy vào cấu hình tenant, bạn chọn chế độ phù hợp.

🔑

Client Credentials

Dùng client_id + client_secret. Phù hợp cho M2M automation, không cần user tương tác.

👤

Password Grant

Dùng username + password. Phù hợp khi bạn có tài khoản SAP BTP cá nhân.

🎫

Bearer Token

Dùng token có sẵn (nhập tay). Phù hợp cho testing hoặc token ngắn hạn.

Luồng hoạt động

Khi bạn gọi một MCP tool (vd sap_search), hệ thống tự động:

1. Kiểm tra token trong bộ nhớ (memory cache) 2. Nếu hết hạn → kiểm tra secret đã mã hóa trong secrets.json 3. Nếu còn hạn → dùng lại (tránh gọi OAuth2 liên tục) 4. Nếu hết hạn → gọi OAuth2 endpoint để lấy token mới 5. Tự động lưu refresh_token + expiresAt vào secrets.json 6. Token được refresh 30 giây trước khi hết hạn (SAFETY_MARGIN_S)

Cấu hình OAuth2 Client trên SAP BTP

Trước khi setup, bạn cần tạo OAuth2 client trên SAP BTP cockpit:

  1. Đăng nhập SAP BTP Cockpit → vào Subaccount của bạn
  2. Vào menu Security > OAuth2 Client (hoặc API Clients trong các phiên bản mới)
  3. Tạo client mới với Grant Type = Client Credentials
  4. Ghi lại Client ID và Client Secret
  5. Nếu dùng IAS (Identity Authentication), lưu ý token URL là /oauth2/token thay vì /oauth/token

Token Lifecycle & Auto-Refresh

Agent quản lý vòng đời token hoàn toàn tự động:

🔄 Tự động refresh

Token được cache trong bộ nhớ. Khi còn 30 giây trước khi hết hạn, agent tự động gọi OAuth2 endpoint để lấy token mới — bạn không cần làm gì cả.

💾 Lưu token an toàn

Token mới + refresh_token + expiresAt được lưu vào secrets.json đã mã hóa. Khi restart agent, token được phục hồi từ file mà không cần xác thực lại.

🔐 Mã hóa đa nền tảng

Windows: DPAPI (gắn với tài khoản Windows). macOS/Linux: AES-256-GCM, key derive từ hostname + username qua scrypt. File mode 0o600.

🍪 Cookie Auth & Web Popup Re-Auth

Bên cạnh OAuth2, SAP ABAP Agent hỗ trợ cookie-based authentication sử dụng SAP session cookies (MYSAPSSO2, SAP_SESSIONID, sap-usercontext). Phương thức này đặc biệt hữu dụng khi bạn đã có sẵn session từ SAP GUI, Fiori Launchpad, hoặc muốn tận dụng cơ chế Single Sign-On (SSO).

⚡ Tính năng chính

🍪

Cookie-Based Auth

Dùng session cookies SAP thay vì OAuth2 tokens. Hỗ trợ MYSAPSSO2, SAP_SESSIONID, v.v.

🖥️

Web Popup Re-Auth

Khi session hết hạn (401), tự động mở browser cho user đăng nhập lại — lấy cookies mới, tiếp tục công việc.

🛡️

Stampede Protection

Nhiều request cùng bị 401 chỉ kích hoạt 1 lần re-auth (mutex + cooldown 5s) — giống cơ chế reauthMu của vibing-steampunk.

🔒

Lưu cookies an toàn

Cookies được mã hóa và lưu trong secrets.json (DPAPI / AES-256-GCM), phục hồi khi restart agent.

🌐 Web Popup Re-Auth — Khi Session Hết Hạn

Khi đang code mà session SAP hết hạn (HTTP 401), agent tự động:

1. Phát hiện 401 Unauthorized từ SAP 2. Kích hoạt stampede guard (chỉ 1 lần re-auth dù nhiều request cùng fail) 3. Gọi ReauthHandler — mở trình duyệt web 4. Hiển thị thông báo "🔐 SESSION SAP DA HET HAN!" + URL đăng nhập 5. User đăng nhập vào SAP qua browser 6. Manual mode: User copy cookies từ DevTools (F12) → paste vào terminal 7. Auto mode: Playwright tự động trích xuất cookies sau login 8. Cookies mới được lưu vào secrets.json (mã hóa) 9. Request được retry tự động với cookies mới → thành công!

🧹 Skill Curator — vòng đời skill tự động

Skill tự tạo trong memory/procedural/skills/ càng tích luỹ càng dễ "chết" — không còn ai dùng nhưng vẫn nằm mãi. reference/scripts/skill_curator.py xử lý việc này, đúng cơ chế Curator của Hermes: skill không dùng tới sau 30 ngày chuyển stale, sau 90 ngày chuyển archived (di chuyển vào .archive/, không bao giờ xoá thật). Dùng lại 1 skill sẽ tự đưa nó về active. Chạy tự động mỗi khi sap-daily-learner được gọi — không cần setup gì thêm.

⏰ Cron thật — daily tip chạy nền qua Windows Task Scheduler

⚠️ Mặc định TẮT (opt-in tuyệt đối), cùng triết lý với Continuous Improvement Engine. Mỗi lần cron thật sự chạy sẽ gọi 1 phiên Claude Code (claude -p) thật — tốn chi phí API thật trên tài khoản của bạn. Cài xong vẫn chưa chạy gì cả cho tới khi bạn bật rõ ràng.

Khác Hermes bản gốc (có gateway daemon chạy liên tục, tự poll mỗi 60 giây): plugin này không có tiến trình nền thật — dùng Windows Task Scheduler làm "cái luôn túc trực" thay cho daemon đó.

Bước 1 — Cài lịch Task Scheduler (chưa chạy gì, cần quyền Administrator)

Click phải reference\scripts\install-daily-learner-cron.bat → Run as administrator

Sau khi cài, task sẽ tự "tick" mỗi 5 phút nhưng vẫn no-op (không làm gì) cho tới Bước 2.

Bước 2 — Bật opt-in thật sự

PowerShell Windows
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.sap-abap-agent\cron"
New-Item -ItemType File -Force -Path "$env:USERPROFILE\.sap-abap-agent\cron\ENABLED"
CMD Windows
if not exist "%USERPROFILE%\.sap-abap-agent\cron" mkdir "%USERPROFILE%\.sap-abap-agent\cron"
echo. > "%USERPROFILE%\.sap-abap-agent\cron\ENABLED"

Hoặc set biến môi trường hệ thống SAP_ABAP_AGENT_CRON_ENABLED=1 thay vì tạo file marker — 2 cách tương đương.

python reference/scripts/cron_manage.py status "$(python reference/scripts/agent_home.py)"

🎯 Progressive Learning Paths

Beginner Path (cho người mới — 4 tuần):

Tuần 1: MM (procurement cơ bản) → Purchase Order → Goods Receipt → Invoice Receipt Tuần 2: SD (sales cơ bản) → Sales Order → Delivery → Billing Tuần 3: FI (kế toán cơ bản) → General Ledger → AP → AR Tuần 4: CO (controlling cơ bản) → Cost Center → Internal Order → Product Costing

Intermediate Path (cho người đã biết SAP):

  • Chọn 1 module chuyên sâu + 2 module liên quan
  • Ví dụ: MM advanced + FI (valuation) + SD (stock transport)
  • Ví dụ: PP advanced + MM (BOM/component) + QM (inspection)
  • Ví dụ: FI advanced + CO (cost center) + TR (cash management)

Expert Path (cho chuyên gia — cross-module integration):

🔄 Order-to-Cash (SD → FI → CO) 🔄 Procure-to-Pay (MM → FI) 🔄 Plan-to-Produce (PP → MM → QM) 🔄 Maintain-to-Close (PM → MM → FI → CO) 🔄 Project-to-Report (PS → SD → CO → FI) 🔄 Transport-to-Settle (TM → SD → FI)

⚠️ Lỗi thường gặp

❌ 401 Unauthorized

Client_secret sai hoặc hết hạn. Chạy sap-btp-agent setup <profile-id> để cập nhật thông tin OAuth2.

⚠️ 404 /oauth/token

URL token sai. Vào profiles/<id>/secrets.json, sửa tokenUrl (vd: đổi /oauth/token thành /oauth2/token nếu dùng IAS).

🔒 Không giải mã được secret

Đổi máy / hostname. Chạy lại sap-btp-agent setup <profile-id> để tạo lại secret.

📭 Chưa có profile nào

Chạy sap-btp-agent setup <URL> để thêm project SAP đầu tiên.

🔌 MCP tool không xuất hiện

Kiểm tra đã claude mcp add --transport stdio sap-btp -- sap-btp-agent chưa (claude mcp list để xem). Đảm bảo sap-btp-agent có trong PATH.

💻 'sap-btp-agent' is not recognized

PATH thiếu folder chứa entry point — lỗi rất hay gặp trên Windows khi pip cài vào user-scheme site-packages. Chạy python -m sap_btp_agent.doctor (chạy được dù PATH sai) để tự phát hiện và lấy sẵn lệnh fix.

🐛 Không thấy report lỗi

Error reporter mặc định TẮT (opt-in). Để bật, xem mục Báo cáo lỗi tự động ở trên.

🐙 About GitHub (Actions, Pages, Releases)

GitHub là nơi lưu trữ source code, chạy CI/CD tự động và phát hành bản cập nhật cho plugin này. Nếu bạn chưa từng dùng GitHub, đây là 4 khái niệm cốt lõi bạn sẽ gặp:

📦

Repository (Repo)

Thư mục dự án chứa toàn bộ code, docs, scripts. Repo này: github.com/StormShynn/sap-abap-agent. Bạn có thể xem code, mở Issue báo lỗi, tạo Pull Request đóng góp — tất cả qua trình duyệt, không cần cài gì.

🤖

GitHub Actions (CI/CD)

Workflow tự động chạy mỗi khi có push lên nhánh main. Repo này có 4 workflow: deploy.yml (deploy site lên GitHub Pages), sync-index.yml (đồng bộ version/counts vào index.html), version-bump.yml (auto-bump plugin.json + build wheel + tạo Release), jekyll-gh-pages.yml (build Jekyll).

🌐

GitHub Pages

Dịch vụ host web tĩnh miễn phí từ GitHub. Trang bạn đang đọc chính là GitHub Pages của repo này, URL: stormshynn.github.io/sap-abap-agent. Mỗi lần deploy.yml chạy xong, site tự cập nhật trong vài chục giây.

🏷️

Releases & Tags

Mỗi lần plugin version được bump (v1.3.3, v1.3.2...), CI tạo git tag + GitHub Release kèm file .whl cho MCP server. Claude Code marketplace đọc tag mới → tự nhắc bạn cập nhật plugin. Xem tất cả tại trang Releases.
